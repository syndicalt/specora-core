# Next.js Frontend Generation — Design Spec

**Date:** 2026-04-08
**Status:** Approved
**Issue:** syndicalt/specora-core#13

## Purpose

Generate a complete Next.js 15 frontend from the same contracts that generate the backend. Page contracts define views (table, kanban), entity contracts define fields and form inputs, route contracts define API endpoints, workflow contracts define state machine columns. The generated frontend is a 4th service in the Docker stack.

## Tech Stack

- **Next.js 15** (App Router)
- **shadcn/ui** (component library)
- **Tailwind CSS** (styling)
- **TypeScript** (types already generated by existing TS generator)
- **Lucide React** (icons, from EntityIR.icon)

## Component Details

### API Client (`lib/api.ts`)

Generated from RouteIR. One function per endpoint:

```typescript
const API = process.env.NEXT_PUBLIC_API_URL || "http://localhost:8000";

// From RouteIR for entity/task_manager/task
export const tasks = {
  list: (limit = 100, offset = 0) =>
    fetch(`${API}/tasks/?limit=${limit}&offset=${offset}`).then(r => r.json()),
  get: (id: string) =>
    fetch(`${API}/tasks/${id}`).then(r => r.json()),
  create: (data: any) =>
    fetch(`${API}/tasks/`, { method: "POST", headers: {"Content-Type": "application/json"}, body: JSON.stringify(data) }).then(r => r.json()),
  update: (id: string, data: any) =>
    fetch(`${API}/tasks/${id}`, { method: "PATCH", headers: {"Content-Type": "application/json"}, body: JSON.stringify(data) }).then(r => r.json()),
  delete: (id: string) =>
    fetch(`${API}/tasks/${id}`, { method: "DELETE" }),
  transition: (id: string, state: string) =>
    fetch(`${API}/tasks/${id}/state`, { method: "PUT", headers: {"Content-Type": "application/json"}, body: JSON.stringify({ state }) }).then(r => r.json()),
};
```

### Data Table (`DataTable.tsx`)

Generated from `views[type=table].columns` + EntityIR fields:
- Sortable column headers
- Reference fields show display name (from FieldIR.reference.display), not UUID
- Enum fields render as colored badges
- Boolean fields render as checkmarks
- Date fields formatted for locale
- Click row → navigate to detail page
- Actions column: edit, delete buttons

### Kanban Board (`KanbanBoard.tsx`)

Generated from `views[type=kanban]` + StateMachineIR:
- Columns = workflow states (from StateMachineIR.states)
- Column headers colored by category (open=blue, hold=yellow, closed=green)
- Cards show fields from `views[type=kanban].card_fields`
- Drag card between columns = call transition endpoint
- Only valid transitions allowed (from StateMachineIR.transitions)
- Terminal states (StateMachineIR.states[x].terminal) have no outgoing drag

### Entity Form (`EntityForm.tsx`)

Generated from EntityIR.fields:

| FieldIR.type | Input Component |
|-------------|----------------|
| string | `<Input>` |
| text | `<Textarea>` |
| integer | `<Input type="number" step="1">` |
| number | `<Input type="number">` |
| boolean | `<Switch>` |
| email | `<Input type="email">` |
| date | `<DatePicker>` (shadcn) |
| datetime | `<DatePicker>` with time |
| enum | `<Select>` with options from FieldIR.enum_values |
| uuid (with reference) | `<Combobox>` (search picker, fetches from referenced entity's list endpoint) |
| array | `<TagInput>` (comma-separated) |

- Required fields marked with asterisk
- Immutable fields disabled on edit
- Computed fields hidden on create, shown as read-only on edit
- Validation from FieldIR.constraints (maxLength, min, max, pattern)

### Filter Sidebar (`FilterSidebar.tsx`)

Generated from PageIR.filters or auto-inferred from enum/reference fields:
- Enum fields → multi-select filter
- Reference fields → entity picker filter
- Date fields → date range filter
- Boolean fields → toggle filter

### Detail View (`DetailView.tsx`)

Generated from EntityIR fields + PageIR.sections:
- Header: entity name/number, status badge (from state machine)
- Fields displayed in sections (or auto-grouped by type)
- Reference fields are clickable links to the referenced entity
- State machine widget: current state + transition buttons for valid next states
- Edit button → opens EntityForm
- Delete button with confirmation

### App Sidebar (`AppSidebar.tsx`)

Generated from all PageIR objects in the domain:
- Each PageIR becomes a nav item
- Icon from EntityIR.icon (Lucide icon name)
- Active state based on current route
- Collapsible on mobile

### Dashboard (`page.tsx`)

Generated overview page:
- Card per entity with total count (fetched from list endpoint)
- Recent items per entity (last 5)
- Quick action buttons (create new)

## Docker Integration

Added as 4th service:

```yaml
frontend:
  build:
    context: ./frontend
    dockerfile: Dockerfile.frontend
  ports:
    - "3000:3000"
  environment:
    NEXT_PUBLIC_API_URL: http://app:8000
  depends_on:
    - app
```

Dockerfile.frontend:
```dockerfile
FROM node:20-slim
WORKDIR /app
COPY package.json package-lock.json ./
RUN npm ci
COPY . .
RUN npm run build
CMD ["npm", "start"]
```

## Generator Structure

New generator target: `nextjs`

```
forge/targets/nextjs/
├── __init__.py
├── generator.py         # NextJSGenerator — orchestrates all sub-generators
├── gen_scaffold.py      # package.json, next.config, tailwind config, tsconfig
├── gen_api_client.py    # lib/api.ts from RouteIR
├── gen_components.py    # DataTable, KanbanBoard, EntityForm, etc.
├── gen_pages.py         # App router pages from PageIR
├── gen_layout.py        # Root layout + AppSidebar from all PageIRs
└── gen_docker.py        # Dockerfile.frontend + docker-compose frontend service
```

Registered as `--target nextjs` in the CLI.

## Dependencies on Existing IR

|-----------|----------|
| PageIR.route | App router directory structure |
| PageIR.views | Table columns, kanban config |
| PageIR.data_sources | API client endpoint mapping |
| PageIR.filters | Filter sidebar config |
| EntityIR.fields | Form inputs, table cell rendering, detail view |
| EntityIR.icon | Sidebar navigation icons |
| EntityIR.state_machine | Kanban columns, state transition buttons |
| RouteIR.endpoints | API client function signatures |
| RouteIR.base_path | API URL prefixes |
| StateMachineIR.states | Kanban column names, state badge colors |
| StateMachineIR.transitions | Valid drag targets, transition buttons |
| FieldIR.reference | Combobox/search pickers, linked detail views |
| FieldIR.enum_values | Select options, badge colors |
